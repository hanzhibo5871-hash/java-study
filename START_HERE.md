# 从这里开始

这份项目现在有两条线：

- 学习线：按 6 周文档一步步理解 Java、SQL、Spring Boot。
- 项目线：最终完成一个任务管理后端 API。

如果你刚开始，不要直接打开所有源码。请每天只完成一个小目标。

## 第一次打开项目时做什么

先打开：

```text
docs/week-01-java-object.md
```

第一天只做里面的“第 1 天：看懂一个 Java 类”。

不要急着学数据库，不要急着启动 Spring Boot。你当前最重要的是先理解：

```text
类 -> 对象 -> 字段 -> 方法
```

## 推荐学习节奏

每天 60 到 90 分钟：

```text
10 分钟：读当天目标
25 分钟：看代码和概念解释
35 分钟：自己敲代码或执行 SQL
10 分钟：对照结果要求检查
10 分钟：回答文档里的问题
```

如果当天的“结果要求”做不到，不要进入下一天。

| 周次 | 文档 | 你会学什么 | 过关标准 |

## 你应该怎么判断自己学会了

不要用“我看过了”判断。

用下面标准判断：

- 能不能自己写一遍？
- 能不能运行并看到正确结果？
- 能不能说清楚代码为什么这样写？
- 换一个标题、状态、ID 后，你还能不能改对？
- 报错时，你能不能根据错误提示定位大概原因？

## 第一阶段暂时不要做的事

在完成第 1、2 周前，先不要做这些：

- 不要改 Spring Boot 后端代码。
- 不要研究 JWT。
- 不要背 MyBatis-Plus。
- 不要直接复制接口请求。

你先把控制台项目吃透。Spring Boot 项目只是把同样的任务管理逻辑搬到了 Web 和数据库里。

## 每次卡住时问自己

```text
1. 这个变量现在保存的是什么？
2. 这个对象是在哪里创建的？
3. 这个方法是谁调用的？
4. 这条数据现在保存在内存里，还是数据库里？
5. 当前代码是在接收输入、处理业务，还是输出结果？
```

带着这些问题读代码，会比从头硬啃更容易。
